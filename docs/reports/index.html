Engram 深入解析报告

一座云端多租户 agent 记忆系统的完整剖析 —— 从内容寻址对象到后台维护管线。

基于 main 分支 · L1–L5d 全部实现并合并 · 生成于 2026-06

Engram 把一套 git 风格的「记忆文件系统」搬到服务端：内容寻址的不可变对象 + Postgres 上对 agent_id→HEAD 的 CAS + 无状态计算。每个 agent 拥有一个版本化的记忆仓库，由无状态 worker 服务；agent 能读取常驻记忆、按需召回长尾、跨会话持久地编辑记忆。设计刻意轻量——不上 Temporal、不上 Kafka——一切并发与一致性都汇聚到那一个 ref CAS 上。

核心一句话：记忆是主动式（agentic）的，不是 RAG。推理前只装载常驻 system/ + 一份廉价的树索引（文件名 + 描述）；长尾由模型在循环中通过 recall/read 工具自行拉取。
